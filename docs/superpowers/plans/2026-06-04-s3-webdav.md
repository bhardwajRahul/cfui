# S3 WebDAV Implementation Plan

Date: 2026-06-04

## Goal

Ship generic S3 WebDAV with multiple mount configurations, Cloudflare R2 as an optional provider preset, web UI file management, and a dedicated `s3_webdav_settings` table.

## Tasks

- [x] Replace R2-only config with `S3WebDAVConfig` and per-mount `S3WebDAVMountConfig`.
- [x] Add Ent schema and generated code for `s3_webdav_settings`.
- [x] Store global feature state in `app_settings` and mount rows in `s3_webdav_settings`.
- [x] Replace `internal/r2dav` with `internal/s3dav`.
- [x] Replace `/api/r2/*` routes with `/api/s3/*` routes.
- [x] Mount WebDAV at `/webdav/` and dispatch by longest matching mount path.
- [x] Support custom mount paths such as `/webdav/my_r2/` and `/webdav/qiniu/`.
- [x] Add S3 connection testing for the selected mount.
- [x] Keep Cloudflare API Token checks out of S3 WebDAV enablement.
- [x] Support optional R2 bucket list/create only in Cloudflare R2 provider mode.
- [x] Rework the frontend into a multi-mount S3 WebDAV screen.
- [x] Hide the S3 WebDAV tab when the feature flag is disabled.
- [x] Remove old R2-only schema, service, routes, scripts, and UI references.
